How to Upload Your WSJT-X Data Files for Visualization

To upload your data fies, you need to download and run the uploader.

Pre-reqs:

  • The uploader requires Java SE 8. Download from here http://www.java.com/en/ if you don't already have installed.

Download:

  • Download SpotvizLogUploader-1.0.jar from the release page: https://github.com/kevinhooke/WSJTLogParser/releases/tag/1.0
  • Save the downloaded jar to a folder locally

To run:

  • Start a command prompt/shell/terminal, and from the command line cd into the folder containing the downloaded jar and then type:
java -jar SpotvizLogUploader-1.0.jar --callsign YOUR_CALL --file /PATH/TO/WSJT-X/ALL.TXT

Where:

  • YOUR_CALL is your callsign - this will be used to store your uploaded data against, so when searching for your call you can find your uploaded data.
  • /PATH/TO/WSJT-X/ALL.TXT is the path to WSJT's ALL.TXT file (see below for example locations depending on your platform.

The parser runs once a minute on the minute to sync with new spots being logged to WSJT-X's ALL.TXT file. If you leave it running, it will continue to watch the file for new updates and upload them every minute. Otherwise you can run it one time, leave for a minute to ensure it runs, and then Ctrl-C to end it.

Path to WSJT-X ALL.TXT file

Depending on the version of WSJT-X and the operating system you are running on, your ALL.TXT data file will be in one of the following locations:

WSJT-X on Mac OS X

WSJT-X versions 1.3 and earlier

Assuming you installed WSJT-X to the /Applications/ folder, use:

  • --file "/Applications/WSJT-X/ALL.TXT"

WSJT-X versions 1.4 and later

In 1.4, the ALL.TXT is stored by default in your User folder, use:

  • --file "/Users/USER_ID/Library/Application Support/WSJT-X/ALL.TXT"

where USER_ID is your Mac user account that you logon as when running WSJT-X. If in doubt, open a Terminal and type 'ls /Users' to find the name of your user home directory.

WSJT-X on Windows 7 and 8.x

Use:

  • --file "C:/Users/USER_ID/AppData/Local/WSJT-X"

where USER_ID is your Windows user account that you logon as when running WSJT-X.
